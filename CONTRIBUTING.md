# Contributing to Lovia

Thank you for your interest in contributing to Lovia!  

> **Note:** Lovia is **source-available** and **not open-source**.  
> Commercial use, redistribution, or cloning for profit is strictly prohibited.  
> Contributions are welcome **only for educational purposes or learning**.

---

## Reporting Bugs

If you find a bug, please:

1. Check if an issue already exists.
2. If not, open a new issue with:
   - A clear title
   - Steps to reproduce
   - Screenshots (if applicable)

---

## Feature Requests

If you have a suggestion for improvement:

1. Open an issue
2. Describe your idea and why it would improve Lovia
3. Keep in mind that only educational or learning-focused contributions are accepted

---

## Code Contributions (for learning purposes)

1. Fork the repository
2. Create a feature branch: `git checkout -b my-feature`
3. Make your changes
4. Test locally (`npm run dev`)  
5. Open a Pull Request describing your changes

> PRs will be reviewed, but only accepted for educational examples or feedback. Commercial or production use is **not allowed**.

---

## Other Ways to Help

- Suggest improvements via issues  
- Give feedback on features or UX  
- Share Lovia with others for educational purposes
